Auxiliar de trabajo: Tareas para modelado de datos

Tareas para modelado de datos

Propósito: Use este auxiliar de trabajo para ayudarle con la identificación de tareas básicas implicadas en el modelado de datos.

Tareas en el proceso de modelado de datos

TareaDescripción
Identificar tipos de entidades o entidadesUn tipo de entidad describe un grupo de objetos, como por ejemplo un grupo de cosas, lugares o pedidos. No obstante, cada entidad tiene su propio concepto.
Identificar atributosLos tipos de entidades pueden tener más de un atributo. Considere columnas de datos; una columna representa la aplicación de un atributo de datos.
Aplicar convenciones de nomenclaturaLos modelos lógicos proporcionan legibilidad de usuario final, mientras que la convención de nomenclatura física tiene en cuenta especificaciones técnicas. Deben estar disponibles a través de la empresa directrices y estándares de nomenclatura que cumplan protocolos de codificación de su organización.
Identificar relacionesLas entidades tienen relaciones entre sí. Por ejemplo, un cliente vive en una dirección, usa un número de cuenta bancaria y tiene un número de la Seguridad Social. "Vive en," "usa," y "tiene" describen relaciones entre entidades.
Aplicar patrones de modelo de datosLos modelos de datos tienen patrones comunes que pueden reutilizarse a través de varias plataformas. Independientemente de que los datos traten sobre una persona o producto, la clasificación del patrón sigue siendo la misma.
Asignar clavesLa clave natural hace referencia a uno o más atributos particulares para el concepto de negocio. La clave subrogada, que es solo una nueva columna, hace referencia a una clave sin valor específico.
Reducir redundancia de datosLos desarrolladores de aplicaciones se centran en la normalización de datos, una técnica para organizar atributos de datos dentro de un modelo de datos para que los tipos de entidades estén más conectados en lugar de almacenados en varios lugares de una base de datos.
Mejorar el rendimiento

Para acelerar un proceso y mejorar el rendimiento, es importante desnormalizar algunas partes del esquema de datos.
